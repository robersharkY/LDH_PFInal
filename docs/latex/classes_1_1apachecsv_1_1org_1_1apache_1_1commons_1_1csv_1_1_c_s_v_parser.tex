\doxysubsection{Detailed Description}
Parses CSV files according to the specified format.

Because CSV appears in many different dialects, the parser supports many formats by allowing the specification of a {\ttfamily \mbox{\hyperlink{classes_1_1apachecsv_1_1org_1_1apache_1_1commons_1_1csv_1_1_c_s_v_format}{CSVFormat}}}.

The parser works record wise. It is not possible to go back, once a record has been parsed from the input stream.

\doxysubsubsection*{Parsing into memory}

If parsing record wise is not desired, the contents of the input can be read completely into memory. 


\begin{DoxyPre}
Reader in = new StringReader("{}a;b\(\backslash\)nc;d"{});
\mbox{\hyperlink{classes_1_1apachecsv_1_1org_1_1apache_1_1commons_1_1csv_1_1_c_s_v_parser}{CSVParser}} parser = new \mbox{\hyperlink{classes_1_1apachecsv_1_1org_1_1apache_1_1commons_1_1csv_1_1_c_s_v_parser}{CSVParser}}(in, \mbox{\hyperlink{classes_1_1apachecsv_1_1org_1_1apache_1_1commons_1_1csv_1_1_c_s_v_format_a6a430701b467d522bb292b5148e8f3e2}{CSVFormat.EXCEL}});
List<\mbox{\hyperlink{classes_1_1apachecsv_1_1org_1_1apache_1_1commons_1_1csv_1_1_c_s_v_record}{CSVRecord}}> list = parser.getRecords();
\end{DoxyPre}


There are two constraints that have to be kept in mind\+: 


\begin{DoxyEnumerate}
\item Parsing into memory starts at the current position of the parser. If you have already parsed records from the input, those records will not end up in the in memory representation of your CSV data. 
\item Parsing into memory may consume a lot of system resources depending on the input. For example if you\textquotesingle{}re parsing a 150MB file of CSV data the contents will be read completely into memory. 
\end{DoxyEnumerate}
