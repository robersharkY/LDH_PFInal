\doxysubsection{Detailed Description}
Apache Commons CSV Format Support.

CSV are widely used as interfaces to legacy systems or manual data-\/imports. CSV stands for \char`\"{}\+Comma Separated Values\char`\"{} (or sometimes \char`\"{}\+Character Separated    Values\char`\"{}). The CSV data format is defined in \href{http://tools.ietf.org/html/rfc4180}{\texttt{ RFC 4180}} but many dialects exist.

Common to all file dialects is its basic structure\+: The CSV data-\/format is record oriented, whereas each record starts on a new textual line. A record is build of a list of values. Keep in mind that not all records must have an equal number of values\+:


\begin{DoxyPre}
      csv    := records*
      record := values*
\end{DoxyPre}


The following list contains the CSV aspects the Commons CSV parser supports\+:


\begin{DoxyDescription}
\item[Separators (for lines) ]The record separators are hardcoded and cannot be changed. The must be \textquotesingle{}\textbackslash{}r\textquotesingle{}, \textquotesingle{}~\newline
\textquotesingle{} or \textquotesingle{}\textbackslash{}r~\newline
\textquotesingle{}.


\item[Delimiter (for values) ]The delimiter for values is freely configurable (default \textquotesingle{},\textquotesingle{}).


\item[Comments ]Some CSV-\/dialects support a simple comment syntax. A comment is a record which must start with a designated character (the comment\+Starter). A record of this kind is treated as comment and gets removed from the input (default none)


\item[Encapsulator ]Two encapsulator characters (default \textquotesingle{}\char`\"{}\textquotesingle{}) are used to enclose -\/\texorpdfstring{$>$}{>} complex values.


\item[Simple values ]A simple value consist of all characters (except the delimiter) until (but not including) the next delimiter or a record-\/terminator. Optionally all surrounding whitespaces of a simple value can be ignored (default\+: true).


\item[Complex values ]Complex values are encapsulated within a pair of the defined encapsulator characters. The encapsulator itself must be escaped or doubled when used inside complex values. Complex values preserve all kind of formatting (including newlines -\/\texorpdfstring{$>$}{>} multiline-\/values)


\item[Empty line skipping ]Optionally empty lines in CSV files can be skipped. Otherwise, empty lines will return a record with a single empty value. 
\end{DoxyDescription}

In addition to individually defined dialects, two predefined dialects (strict-\/csv, and excel-\/csv) can be set directly.

Example usage\+:

\begin{quote}

\begin{DoxyPre}
Reader in = new StringReader("{}a,b,c"{});
for (\mbox{\hyperlink{classes_1_1apachecsv_1_1org_1_1apache_1_1commons_1_1csv_1_1_c_s_v_record}{CSVRecord}} record : CSVFormat.DEFAULT.parse(in)) \{
    for (String field : record) \{
        System.out.print("{}\(\backslash\)"{}"{} + field + "{}\(\backslash\)"{}, "{});
    \}
    System.out.println();
\}
\end{DoxyPre}
\end{quote}
